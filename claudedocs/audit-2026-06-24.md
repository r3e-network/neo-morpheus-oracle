## Real bugs fixed

1. **#16 — Rich-callback dispatch:** Consumers implementing `onMiniAppResult` were never invoked by the kernel; they silently received `appId="legacy"`/`requester=null` via the legacy path. Now dispatches the rich callback as designed, with backward-compatible `onOracleResult` fallback for already-deployed legacy consumers.
2. **#10 — Producer/verifier CBOR disagreement:** The enclave silently produced empty bstrs for indefinite-length CBOR (`length=-1` → empty `subarray`), while the relayer threw `unsupported minor 31`. The two sides disagreed. Unified into one shared codec with correct RFC 8949 indefinite-length handling.
3. **#9 — NeoX deadline-timer hang:** `setTimeout(...).unref()` on the receipt-deadline timer meant it never fired on an idle event loop, so a stuck `tx.wait()` hung forever instead of failing fast. `unref()` removed; `finally { clearTimeout }` still bounds lifetime.
4. **#8 — Grant-key drift hazard:** The consensus-critical `keccak256(appId|"|moduleId)` derivation was inlined at three call sites (write/check/read). Single-sourced into `_grantKey`, eliminating the "write key A / read key B" authorization-bug class.

## Real performance wins

1. **#15:** Eliminates one full `Storage.Get` + `StdLib.Deserialize` of a 9-field `MiniAppRecord` on every `SubmitMiniAppRequest` — the hottest write path. Direct GAS saving.
2. **#14:** Per-block N+1 serial `getapplicationlog` → bounded-concurrency fan-out (≤4 in-block × ≤8 block-level = ≤32 concurrent). Real latency win for multi-tx blocks.
3. **#13:** Removed redundant full-array `map` + alias resolution on the hot ZK verify path (call count unchanged, front-loaded work halved).
4. **#12 / #8 / #11:** Dedup (digest computation, hand-copied `Request` tuple, triplicated RPC snippets) — maintainability + indirect perf.

## What was checked, domain by domain

### #8 — EVM contracts (`contracts-evm/*.sol`)

- **`Request` tuple layout (critical):** `Request` is stored in a `mapping(uint256 => Request)` and tuple-decoded by `MiniAppDiceGameEVM.getRequest()`. Verified the 15-field order is byte-identical pre/post (`id, appId, moduleId, operation, payload, requester, callbackContract, status, createdAt, fulfilledAt, success, result, error, feePaid, feePayer`). The PR only _moved_ the struct into a shared `IMorpheusOracleEVM.sol`; no reorder/insert. No storage or ABI break.
- **`_grantKey`:** `internal pure`, no storage/`msg.sender`/privilege touches. Byte-identical to all three inlined originals. Events and storage writes remain at original call sites.
- **Interface additions:** purely additive (`requestFromCallback`, `getRequest` already existed in the kernel; now `override`). No selector change.
- **`MiniAppDiceGameEVM` (−39):** removed a hand-copied interface block; `settleFromKernel`'s `r.status == 2 || 3` → named `IMorpheusOracleEVM.Status.Succeeded/Failed` (Succeeded=2, Failed=3 — identical).

### #16 / #15 — Neo contracts (`contracts/MorpheusOracle/*.cs`)

- **#16 arg order:** kernel `Contract.Call` passes `(requestId, appId, moduleId, operation, requester, success, result, error)`; consumer `OnMiniAppResult` declares the same 8 params in the same order with matching types. Legacy `OnOracleResult(requestId, operation, success, result, error)` likewise matches.
- **#16 fallback:** legacy call lives strictly inside the `catch` of the rich call → EITHER/OR, never both. Neo VM `try/catch` rolls back the failed rich invocation fully, so the consumer is never left partial. The broad `catch` is the only option on Neo (no exception subtypes), and the ordering (rich-first) is the safe choice.
- **#16 artifacts:** exactly 2 lines changed (manifest + NEF/checksum). Method-name set byte-identical (146 names). No storage-prefix change.
- **#15 duplicate read:** both reads hit `AppMap().Get(appId)` with the same key; no write between them. Dropped `app.Active`/`module.Active` asserts ran strictly after `RequireActiveMiniApp`/`RequireActiveModule`, which assert the same thing earlier. All other validation (operation/payload length, grant check) intact.

### #10 / #12 — shared codec (`packages/shared/src/cbor.js`, `workers/morpheus-relayer/src/*`)

- **#10 raw bytes:** `decodeCoseSign1` returns `subarray` slices of the original wire bytes wrapped in `Buffer.from` (faithful copy). The decoded payload _map_ is returned separately and never re-serialized into the verify bytes. `buildCoseSign1SigStructure` only prepends a CBOR length head to the bstr — no inner re-encode. So `crypto.verify('sha384', …)` runs over the exact bytes. Output byte-identical to the old definite-length path.
- **#10 indefinite-length:** correctly handles `0x5f/0x7f/0x9f/bf` with a `BREAK` sentinel; rejects int/tag indefinite per RFC 8949. Fixes two real prior bugs (silent-empty on enclave, throw on relayer).
- **#10 single source:** both `attestation.js` and `enclave-server.mjs` import the shared codec; grep confirms no remaining local decode definitions.
- **#12 digest binding:** new `resolveDigestBinding` is called by both sign and recompute paths; body identical to both removed inline blocks. Same `resolveFulfillmentSigningContext`, same `if (chain === 'neo_n3')`, same `contractScriptHash`/`networkMagic` assignment. Pure move; hash inputs unchanged; no validation dropped.

### #13 / #14 — workers (`workers/*/src/*.js`)

- **#13 groth16:** `verifyGroth16Proof` awaited exactly once both before and after. `normalizePublicSignals` (pure, idempotent) hoisted from 2 call sites to 1. Throw order preserved (normalize still precedes size-limit). No error newly swallowed.
- **#14 applog:** `txHashes = transactions.map(txid||hash).filter(Boolean)` == old `if(!txHash) continue`. `mapWithConcurrency` writes `results[index]` by input index → output order == input order. First RPC error rejects the whole scan (fail-fast, cursor doesn't advance). No shared mutable state raced (the `promoteNeoN3RpcUrl` write is pre-existing and benign). Covered by an explicit equivalence test (`scanNeoN3OracleRequests batched scan is equivalent to the sequential path`) and a fail-fast test.

### #9 / #11 — web/app

- **#9 substance:** only 3 commits carry logic — (a) mainnet oracle catalog refresh, (b) lockfile ethers cleanup, (c) NeoX `unref()` removal. The 4th commit (83 files) is pure prettier. Spot-checked `apps/web/lib/*.ts` and `apps/web/app/api/**/route.ts`: all whitespace-only.
- **#11 snippets:** both `buildNeoRequestInvoke` and `buildCallbackQueryTemplate` reproduce the inline bodies verbatim (field order, method names, type tags, 2-space indent). 6 snippets before, 6 after, zero drops. `neo-snippets.test.ts` asserts verbatim multi-line string snapshots + structural param-slot pinning.
