# Refactoring Report — neo-morpheus-oracle

This report synthesizes 93 adversarially-verified refactoring findings across the Neo N3 C# contracts, the Solidity EVM mirror, the Node relayer, the Nitro TEE worker, the Next.js web app, and the shared ops scripts. Findings are prioritized by leverage (impact vs. effort), then grouped by subsystem. Cross-cutting themes that repeat across subsystems are called out separately, since several individually-small findings compound into systemic risk classes.

A recurring honest caveat applies throughout: many of these are framed as "behavior-preserving" but carry a subtle behavior change at one or two call sites (usually a `trimString` coercion difference, an env-alias precedence swap, or a digest-byte change). Those are flagged per item. Performance claims that depend on storage/GAS pricing or network RTT should be benchmarked against the real backend before being treated as quantified wins — none of the numbers below are measured.

## Tier 1 — Quick wins (meaningful impact, small effort)

These are the highest-leverage changes: they remove a correctness hazard or a measurable cost with low blast radius. Do these first.

### Contracts (Neo N3)

- **Eliminate the double `MiniAppRecord` read+deserialize on request submit.** `MorpheusOracle.Requests.cs:15-18` + `MorpheusOracle.Storage.cs:106-115`. Have `ValidateRequestInputs` return the already-fetched `MiniAppRecord`/`SystemModuleRecord` so `SubmitMiniAppRequestInternal` stops calling `RequireActiveMiniApp` a second time; drop the redundant `app.Active`/`module.Active` asserts the `Require*` helpers already enforce. This is the single most-called write path (every submit funnels through it). Removes one `Storage.Get` + full deserialize of a 9-field record per submission. **Impact medium / effort small.** Risk negligible — same revert messages preserved, no `AppMap` write between the two reads.

- **Collapse the redundant `FeedMap().Get` + `GetLatest` double read in the feed write path.** `MorpheusDataFeed.cs:246-252`. When `existingRaw != null`, deserialize it directly instead of calling `GetLatest(pair)`. This is the hottest write loop in the data-feed contract (every round, every pair). **Impact medium / effort small.** No correctness risk; note the rationale's "~32768 GAS" figure is the `Storage.Put` price, not the `Get` — the actual saving is a cheaper read, and per-iteration cost stays dominated by the `Put`.

- **Reuse the in-hand result hash instead of recomputing `ComputeResultHash` twice on fulfill.** `MorpheusOracle.cs:1139-1185` + `MorpheusOracle.Fulfillment.cs:15-23`. Hoist a single `ComputeResultHash(result ?? "")` and feed both the digest assembly and the `OnMiniAppRequestCompleted` event. Removes one redundant SHA256 over up-to-4KB per fulfillment. **Impact low / effort small.** Do not touch `ComputeFulfillmentDigest`'s signature (security-sensitive signed digest) — use a plain local hoist.

### Contracts (EVM)

- **Extract a single `_grantKey` helper** (`keccak256(abi.encodePacked(appId, "|", moduleId))`) used by `grantModule` (`:184`), `_submit` (`:215`), and `isModuleGranted` (`:339`) in `MorpheusOracleEVM.sol`. An internal `pure` helper inlines at `optimizer_runs=200`, so gas is unchanged; the value is collapsing three independently-editable copies of a security-relevant key derivation, removing a class of "write key A, read key B" authorization bugs. **Impact medium / effort small.** Must keep the literal `"|"` separator and arg order so existing on-chain grants still resolve. (Three near-duplicate findings describe this; treat as one task.)

- **Extract a shared `IMorpheusOracleEVM.sol` interface + `Request` struct** to kill the hand-synced ABI tuple copied into `MiniAppDiceGameEVM.sol:10-37` and `MiniAppMessageEVM.sol:18-23`. Solidity tuple ABI is positional, so a field reorder in the kernel currently corrupts the dice consumer's `getRequest` decode silently; sharing the type makes drift a compile error. **Impact medium / effort small.** If the shared struct adopts the `Status` enum, the dice game's `r.status == 2 || r.status == 3` numeric checks must become enum comparisons; preserve `payable` on `requestFromCallback`. This is also the prerequisite for any later struct-packing work.

### Relayer

- **De-duplicate the retry-delay jitter computation** shared by `fulfillment.js:203-214` (`computeRetryDelayMs`) and `state.js:420-428` (`scheduleRetry`). Move the pure function to a neutral `lib/retry.js`; both paths import it. This is the exponential-backoff-with-jitter policy that protects RPC/Nitro/Supabase from retry stampede — two copies will drift. **Impact medium / effort small.** Keep both existing jitter test suites (`fulfillment.test.mjs` and `state.test.mjs:94-121`); do not import back into `fulfillment.js` (would risk an ESM cycle).

- **Drop the redundant deep clone in `sanitizeForPostgres` before `JSON.stringify`.** `persistence.js:29-42`. Fuse NUL-stripping into a `JSON.stringify` replacer in one pass, deleting the throwaway deep copy allocated on every POST/PATCH. **Impact low / effort small.** Wire output identical; the standalone unit test at `relayer.test.mjs:1939` must be rewritten or removed alongside.

- **Centralize the nitro URL fallback ladder** so the `decryptUrl` never-failover invariant is enforced once, not hand-copied across `config.js:437-477`. A single `resolveEnclaveUrls(...)` returning `{signerUrl, enclaveFulfillUrl, decryptUrl}` makes the confidentiality-boundary precedence (`explicitDecrypt || explicitFulfill || explicitSigner || apiUrl`) a single reviewable expression with a unit-testable seam. **Impact medium / effort small.** Must preserve the comma-joined string semantics of `resolveNitroApiUrls` — a "single URL" helper that did `.split(',')[0]` would break failover for the signer/fulfill lanes.

### Nitro worker

- **Fetch provider quotes for a symbol concurrently** in `feeds.js` `fetchPriceQuotes` (`:432-438`) via `Promise.allSettled` over `resolveQuoteForProvider`. Providers are independent I/O with per-provider circuit breakers and in-flight dedup already in place, so serializing them is pure additive latency. **Impact medium / effort small.** Map provider by array index when partitioning rejections so the per-provider error tag isn't lost; `allSettled` preserves order and aggregation is order-independent.

- **Adopt the existing `buildLaneSignedEnvelope` helper in all fulfillment lanes** (`chain/signing.js:257`), replacing the hand-spread envelope copies in `feeds.js`, `fetch.js`, `compute/index.js`, `paymaster/index.js`. The signed-result envelope is the on-chain verification contract; six copies mean a field addition silently diverges. **Impact medium / effort small.** The four mechanical files are a proven no-op; leave `neodid/index.js` as-is unless its added top-level `signature`/`public_key` is a deliberate decision (no envelope tests exist, so review the diffs by hand).

- **Hoist duplicated `getSupabaseRestConfig` into `platform/core.js`** (copied verbatim in `oracle/crypto.js:333-350`, `providers.js:159-176`, `feeds/feed-state.js:26-43`). The Supabase secret-key fallback order is a security-relevant config contract; one definition makes credential precedence auditable. **Impact medium / effort small.** `feed-state.js` currently `export`s it — preserve via re-export. (This pairs with the relayer/scripts `getSupabaseRestConfig` duplication under the config-sprawl theme.)

### Shared scripts

- **Delete the divergent `parseGasToRaw` copy** in `fix-mainnet-morpheus-runtime.mjs:69-75` and import the tested exact-BigInt version from `lib-gas-units.mjs:23-34`. Two money-parsers that disagree on `+`-prefixed / high-precision input is a correctness hazard on a path that gates mainnet spend. **Impact medium / effort small.** Not strictly behavior-preserving for malformed operator-supplied thresholds (they'd parse instead of falling back) — that shifts toward the intended tested semantics; make the optional `>8-decimal` pre-check mandatory if the stricter rejection must be preserved exactly.

- **Import the canonical confidential-envelope module** in `runtime-service-matrix.mjs:797-856` (and the two checkly sync scripts) instead of re-vendoring X25519-HKDF-AES-GCM. The wire format must stay byte-identical to the worker decryptor, and the hand-rolled copies are not covered by the golden-vector test. **Impact medium / effort small.** Use the existing direct relative import — do **not** add a `package.json` exports subpath (the finding suggesting that is spurious). The canonical function's strict 32-byte key check throws earlier on a bad key (an improvement). Keep `JSON.stringify` at call sites; the `webcrypto` import is still used elsewhere — only the three envelope constants are removable.

- **Consolidate `normalizeHash160` onto `lib-neo-signers.mjs:207-215`**, removing the verbatim copies in `fix-mainnet-morpheus-runtime.mjs:46-54` and `verify-morpheus-n3.mjs:32`. This decides which contract/address a mainnet maintenance script targets. **Impact medium / effort small.** Critical: do **not** delete `strip0x` from `fix-mainnet-morpheus-runtime.mjs` — it has ~8 other call sites on the witness/signer path and removing it is a `ReferenceError`. Only remove `strip0x` in `verify-morpheus-n3.mjs` where it's used solely by `normalizeHash160`.

## Tier 2 — Strategic (high impact, larger effort)

Higher blast radius or larger surface area, but they close systemic correctness gaps. Sequence these deliberately, with tests landing alongside (not after).

### Relayer

- **Extract the CBOR/COSE attestation codec into one shared module** used by both the relayer (`attestation.js:21-113`) and the enclave server (`deploy/nitro/enclave-server.mjs:126-220`). The enclave _produces_ the attestation document and the relayer _verifies_ it; today the relayer **throws** on indefinite-length CBOR (minor 31) while the enclave **silently misdecodes** it — a real divergence that downgrades trust on legitimately-attested results. **Impact high / effort medium.** Behavior-changing by design: a correct shared decoder makes both succeed on indefinite-length AWS NSM docs. `packages/shared` already serves both ESM contexts. Keep `decodeCoseSign1` (full struct, relayer) vs `decodeCoseSign1Payload` (payload only, enclave) as thin wrappers over one `cborRead`. Pin behavior with a shared fixture test.

- **Unify the two divergent Neo N3 `fulfillRequest` submit paths** behind `buildSignAndBroadcastNeoN3Tx` (`neo-n3.js:1030-1082` vs `896-924`/`668-726`). Today fee shaping / top-up / witness verification silently diverge by which key is configured: a locally-signed callback can be broadcast without the 20% callback-gas headroom the runtime path adds, so a cold-storage `onOracleResult` can run out of gas on the local path but not the derived one. Replace try/catch-as-control-flow with up-front adapter selection. **Impact medium / effort large.** This is behavior-changing (local path gains headroom + padding + an auto-top-up funder dependency) and needs fee-math regression coverage. **Land the fulfill-path refactor independently of the queue path** — applying the same seam to `queueNeoN3AutomationRequest` would bypass allowlist/intent-policy checks enforced in the nitro-worker package (a security regression).

- **Replace `prepareOracleFulfillment`'s per-lane if-ladder with a lane handler registry** (`fulfillment.js:891-1186`, 7 branches). Collapse 7 copies of the sign-then-wrap tail into one shared tail, turning lane addition into a table entry. **Impact medium / effort large.** Highest-blast-radius hot path: the unified tail must preserve lane ordering and the `enclaveFulfill && !isHostUnattestedFetchLane` short-circuit, the enclave lane's remote-signature/digest cross-check (must **not** be locally re-signed), per-lane pre-sign transient throws, per-lane `trust_tier` labels, and the decrypt lane's no-`|| ''` forwarding. A smaller, lower-risk first step is the **`signAndPrepare` helper extraction** (separate finding) that single-sources the consensus-critical signing-input shape across the 7 sites without restructuring control flow — do that first.

- **Extract the shared scan-and-process pipeline** from the two `processChain` cursor variants (`request-processor.js:123-235` and `237-313`). ~110 of ~190 combined lines duplicate the most operationally sensitive control flow (cursor advancement, dedup, backpressure, durable persistence). **Impact medium / effort large.** The finding under-states three divergences any merge must preserve: different return shapes (the block variant's `request_reconciliation` is consumed at `relayer.js:60`), an early `return` inside the request variant's `try`, and the block variant's deliberate two-`try/catch` split (cursor advance must stay inside the scan `try`). **The block lane has no direct unit-test coverage** — write tests for `processChain`/`reconcilePendingRequests` _before_ reshaping it through shared code.

### Nitro worker

- **Break `crypto.js` (793 lines) into keystore / confidential-payload / programmable-runtime modules** behind the `oracle/index.js` barrel. `executeProgrammableOracle` (untrusted script/wasm execution) and the sealed-key derivation share no state beyond the key material, yet edits to the riskiest concern currently force re-review of the sandbox and vice versa. **Impact medium / effort medium.** The "importers unchanged" premise is wrong for the 4 importers that bypass the barrel (neodid/compute/fetch/message-reveal) — repoint them or leave a re-export shim. The key-material singleton must stay co-located with both `ensureOracleKeyMaterial` and `decryptEncryptedToken` to avoid duplicate caches; revalidate three test files.

- **Process feed symbols concurrently in `handleOracleFeed`** (`feeds.js:688-843`) by splitting into a parallel fetch phase and the existing synchronous record/decision phase. **Impact medium / effort medium.** Must use a **bounded** concurrency env (`MORPHEUS_FEED_SYMBOL_CONCURRENCY`), not unbounded `Promise.all` over all 33 default symbols — TwelveData would rate-limit (429). Keep plain `Promise.all` (not `allSettled`) so the zero-providers throw still propagates identically. Benchmark against the provider's rate limits before raising concurrency.

### Web

- **Collapse `OracleResponseViewer` (459 lines) and `ComputeOutput` (330 lines) into one parametric output component** plus a reusable `LabeledCodePanel`. ~790 lines of largely-duplicated presentational code; the `aria-live` readiness banner exists in one but not the other. **Impact medium / effort large.** Behavior-preserving only if per-panel divergences are reproduced exactly (text colors, two data-prop adapters, differing `canSubmitWithWallet` default, two CALLBACK READBACK wordings). Update the source-grep test in `oracle-response-viewer.test.ts`. This is the largest single duplication block in the web subsystem but also the riskiest web item — schedule after the smaller web extractions below.

## Tier 3 — Worthwhile (lower impact or medium effort)

Solid maintainability/correctness improvements that are lower-leverage than the tiers above. Batch them opportunistically, ideally alongside related Tier 1/2 work in the same files.

### Relayer

- **Memoize the neo_n3 local verifier account / signer-role resolution** (`fulfillment.js:309-338`, `neo-signers.js:332-446`) to stop re-deriving EC keys per fulfillment. **Impact medium / effort small–medium.** The cache key **must** cover every input `buildRoleReport` reads — including `MORPHEUS_ALLOW_UNPINNED_SIGNERS` (read directly from `process.env`), the full `NEO_N3_SIGNER_ENV_KEYS` set, `config.network`, and `config.neo_n3.updaterPrivateKey/updaterWif`. An incomplete key leaks state across the env-mutating tests and across networks. The per-secret Account `Map` is the lower-risk variant; a process-lifetime report cache keyed only by `(network, role)` would break tests and serve stale signer sets.
- **Cache the ethers Contract + read-RPC URL list** in `neox.js` (`withNeoXReadFailover`/`scanNeoXOracleRequestsById`), extending the existing `providerCache` pattern. Key on `(rpcUrl, chainId, oracleContract)`. Read-only path, so a shared Contract is safe. **Impact low / effort small.**
- **Batch the Neo N3 block-scan `getapplicationlog` calls** under the existing bounded `mapWithConcurrency` (`neo-n3.js:386-416`). Classic N+1 fan-out the block-level path already solved; `mapWithConcurrency` preserves order. **Impact medium / effort medium.** Bound the inner width — total in-flight is `block_width × inner_width`, which can pressure RPC endpoints into rate-limiting.
- **Consolidate `normalizeRequestType`** into `lib/strings.js` (three copies: `router.js`, `automation.js`, `fulfillment.js`). Consensus-adjacent (route + digest input + operator-only gate). **Impact low / effort small.** The only real delta is the `trimString` variant in `fulfillment.js`; keep the strict `lib/strings.js` semantics — do not "harmonize" the two `trimString` implementations.
- **Extract the neo_n3 digest-binding context** shared by the three digest paths (`fulfillment.js:346-365`, `443-457`, `463-481`) so the relayer's own cross-check invariant is enforced by construction, not copy-paste. **Impact low / effort small.** Scope to the n3 binding pair; treat `buildEnclaveFulfillmentContext` as a thin derivation, not a literal shared object.
- **Decouple `processAutomationJobs` orchestration** from the inlined claim/dispatch/persistence steps (`automation.js:699-860`) by extracting `claimNextExecution` and `finalizeExecutionOutcome`. Gives the claim durability invariant a unit-testable seam. **Impact low / effort medium.** Preserve the `inFlightExecution` flag placement between claim and broadcast (governs the stale-reclaim failure lane). Also **factor the duplicated `recordRun` call** in the duplicate-skip vs queued arms (`:808-840`) — one construction site keyed on a computed `status`.
- **Replace text-matching failover/retry classification with structured error types** at the RPC seam (`neox.js` `isNeoXTransportError`, `fulfillment.js` `classifyError`, `neo-n3.js confirmNeoN3FulfillExecution`). A node returning "service temporarily down" instead of "service unavailable" silently flips a transient failure into a permanent on-chain finalization. **Impact medium / effort medium.** The text matchers are load-bearing for _foreign_ errors (ethers codes, `fetch failed`, undici/Supabase) and for errors crossing process boundaries where only `error.message` survives — the fallback can never be deleted, and `instanceof` is fragile under ESM dual-instance. This is heavily characterized by tests asserting exact strings; validate against them.
- **Replace positional destructuring in Neo N3 record decoding** with named field-index maps (`neo-n3.js:235-320`, `340-378`) and single-source the settled/outcome gate. **Impact medium / effort medium.** Part 1 (named maps) is low-risk. **Do not** also drop the `'auto'` encoding sniff in `decodeNeoItem` — `relayer.test.mjs:1882` deliberately asserts the heuristic mis-decodes a hex-look-alike base64, and `neo-n3.js:119-124` documents it as intentional. That half is a behavior change requiring test edits.
- **Split `persistence.js` (713 lines)** into transport / backoff-state / record-shaping modules so the cross-cutting Supabase circuit-breaker (consumed by 3 other modules) is reviewable as a unit. **Impact low / effort medium.** Pure reorg; the one hazard is import direction (`supabase-client.js` must import the backoff marker, not vice versa) and updating ~25 import sites or keeping a re-export shim.
- **Replace order-dependent Supabase error-classifier substring matching** with structured `error.httpStatus` checks (`persistence.js:91-127`) driving the backoff-window selection (quota 5min vs transient 30s). **Impact low / effort small.** Keep the substring fragments as a fallback for errors with no `response` (raw fetch `TypeError`, `AbortError`); the `502/503/504` mapping narrows the connectivity lane vs today, so verify it doesn't regress 5xx codes outside that set.
- **Resolve Supabase network/REST config once per operation** instead of re-reading `process.env` 15+ times (`persistence.js`), routing through the same `env()` resolver `config.js` uses. **Impact low / effort small.** Two honest behavior changes to call out, not assume inert: routing network through `env()` would start honoring `MORPHEUS_RUNTIME_CONFIG_JSON` (latent today — the live renderer emits direct env lines), and unifying the key resolver swaps the 2nd/3rd secret-key alias precedence between `config.js` and `persistence.js`.
- **Unify the two secret-redaction definitions** (`config-schema.js:33-50` and `logger.js:24-29`) behind one canonical fragment list, taking the union so neither sink loses coverage. A key redacted by one but not the other is a latent log leak. **Impact medium / effort small.** Preserve the no-underscore `private_?key`/`api_?key` variants when deriving the regex.
- **Extract the `returnRepresentation → rows[0]` parse block** and the schedulable-claim OR-parts builder shared across the claim helpers (`persistence.js:322-330`, `540-550`). **Impact low / effort small.** Reproduce `buildOrParts`' empty-`dueAtIso`/empty-`staleBeforeIso` edge cases and keep the configurable-vs-hardcoded `staleBeforeIso` per caller.
- **Read the signer-identities registry lazily** rather than at module import (`neo-signers.js:7-11`). **Impact low / effort small.** This is a robustness cleanup (handleable call-site error vs import-time crash on a relocated layout), **not** a performance win — the "blocks the event loop" framing is overstated for a one-time ~2KB sync read.

### Contracts (Neo N3)

The kernel invokes the 5-arg `onOracleResult`; the consumer's rich 8-arg `OnMiniAppResult` (with `requester`/`appId`) can never fire, so consumers built against it silently record `appId="legacy"`, `requester=null`. **Impact medium / effort small–medium.** Naively swapping the `Contract.Call` method name without a fallback breaks every already-deployed consumer that only implements `onOracleResult` — and the `try/catch` swallows it silently. Safe path: deliver `onMiniAppResult` with an `onOracleResult` fallback. `CALLBACK_METHOD` is **not** dead (it's an input-validation whitelist at `:1047/1065/1076`); deleting it requires editing those asserts. Update `OracleCallbackConsumerTest.cs:19-20` and the tracked-fix note in `contracts/README.md:85-88`.
- **Extract a shared request-completion tail** for `FulfillRequest` (`:1139-1185`) and `ExpireStaleRequest` (`:824-861`) — the reserved-fee release + inbox write + completion event guarding the `accrued >= reserved` money invariant. **Impact medium / effort medium.** Do **not** naively hoist the contiguous block: that would fire the external callback on expiry (a real behavior change). Split the seam around the divergent middle; keep the fulfilled-counter increments in `FulfillRequest` only (expiry must not inflate SLA metrics).
- **Bind the data-feed attestation to network + contract** like the sibling contracts (`MorpheusDataFeed.cs:188-233`) by appending `NetworkMagicLe4()` + `ExecutingScriptHash` before `VerifyWithECDsa`, closing a cross-deployment signature-replay gap. **Impact medium / effort small.** Not behavior-preserving — the off-chain feed signer and the test helper must update in lockstep; inert today (no key registered, test harness is the only producer).
- **Share the `NetworkMagicLe4` encoder** instead of duplicating it in `MorpheusOracle.Fulfillment.cs:65-75` and `NeoDIDRegistry.cs:306-316`. A cross-network replay-defense whose bytes must be identical on- and off-chain. **Impact medium / effort small.** Use a `<Compile Link>`'d shared source per `.csproj` (Neo contracts can't cross-call for a pure byte transform); validate the test helper's bit-shift form against passing tests.
- **Batch the redundant global request counters** (`PREFIX_COUNTER` and `PREFIX_TOTAL_REQUESTS` move identically; derive total from the request-id counter). **Impact low / effort small.** On upgrade the old slot becomes orphaned dead storage (harmless). Add a unit test asserting `GetTotalRequests() == max request id`.
- **Paginate the unbounded `GetAll*` enumerators** (`MorpheusOracle.cs:456-478`, `MorpheusDataFeed.cs:278-288/410-420`) with bounded `(startIndex, count)` overloads mirroring `RebuildIndexes`. **Impact medium / effort medium.** Additive/low-risk. Honest correction to the framing: these are `[Safe]` reads via test-invoke RPC, so the ceiling is the node's `maxgasinvoke`/VM limit (call faults/times out), not on-chain tx revert or OOM. Keep `GetAll*` as wrappers so source-assertion tests don't break.

- **Introduce an `IsAdminOrUpdater` / `HasWitness` authorization helper** to dedupe the witness-guard triad across five sites (`MorpheusOracle.cs:769-773`, `Storage.cs:41-57`). **Impact low / effort small.** Outcome-preserving; note `CheckWitness` short-circuiting in `ExpireStaleRequest`'s admin-success path is one fewer syscall (benign, but not byte-identical).
- **Extract the duplicated legacy-alias request preamble** (`MorpheusOracle.cs:1045-1082`, three copies). **Impact low / effort small.** Stop the helper at resolution — the three entrypoints enforce three different authorizations (`CallingScriptHash` vs `ValidateUpdater` vs `CheckWitness`); do not collapse the divergent submit tails.

### Contracts (EVM)

- **Factor `expireStaleRequest`'s mixed concerns** into a shared `_refund` (CEI `call`+`revert RefundFailed`) and `_debitAccrued` helper (`MorpheusOracleEVM.sol:294-329`, overlapping `:151-158`/`:331-334`). Three independently-maintained raw-`.call` refund tails are a CEI/reentrancy footprint. **Impact low / effort small.** The contract is documented bytecode-frozen / next-deployment — this re-opens audit surface on money-movement for a readability gain; the clamp relocation is only equivalent under the `accrued >= reserved >= feePaid` invariant (re-verify it), and `_refund` must replicate the `to != address(0)` guard the other two sites lack.

### Web

- **Deduplicate the 3 RPC reads shared by `/public-key` and `/onchain/state`** by adding `oracleEncryptionPublicKey` to `fetchNeoN3State`'s batch and serving the key block from the 12s-cached state body. **Impact medium / effort medium.** **Not a drop-in removal**: the public-key route carries distinct degraded/error semantics (`available`/`degraded`/`key_source`, `available:false` branch) that `evaluateOracleKeyStatus` keys off and the state body lacks. Removing the second fetch requires extending `OnchainOracleStatus`/`readOracleStateFromBody` and re-deriving the key-status UI + mainnet-degraded path in two clients.

These are the exact strings users paste on-chain; the module's own header says they must stay byte-identical. **Impact medium / effort small.** Have the builder escape internally; a snapshot equality test de-risks it fully.

## Cross-cutting themes

These patterns recur across subsystems; fixing them once per language/runtime pays down a whole risk class rather than a single finding.

1. **Hand-copied security-critical byte formats (digests, envelopes, attestation, replay-binding).** The same wire format is re-implemented in producer and verifier, on-chain and off-chain: the CBOR/COSE codec (relayer vs enclave), the X25519 confidential envelope (canonical module vs 3 scripts), `NetworkMagicLe4` (two contracts), the fulfillment digest binding (3 relayer paths), and the data-feed attestation missing its network binding entirely. These are the highest-stakes duplications — drift produces silent verification failures or weakened replay defenses. Prefer one shared source per boundary with a golden-vector/fixture test binding both ends.

2. **Config/credential-resolution sprawl.** `getSupabaseRestConfig` exists in 3+ copies (relayer, nitro worker ×3), the Supabase network/key resolver diverges from `config.js` (alias-precedence swap), and the nitro URL ladder is hand-copied 3×. Different copies authenticate or route differently per lane, producing "works everywhere except X" bugs. Centralize each resolver and call out the (real) precedence/behavior deltas when unifying.

3. **Repeated RPC / round-trips on hot paths.** N+1 fan-outs and redundant reads recur: serial `getapplicationlog` per tx (relayer), serial provider quotes and serial feed symbols (nitro), double `MiniAppRecord`/`FeedRecord` storage reads (contracts), duplicate `/public-key` + `/onchain/state` fetches (web), and the two-RTT Upstash limiter. Bounded concurrency and read-reuse are the fixes — but every concurrency change needs a rate-limit benchmark, and storage-GAS claims need pricing verification.

4. **Divergent string/normalization helpers as a correctness hazard.** `normalizeRequestType` (×3, routing + digest key), `normalizeHash160`/`normalizePublicKey` (×multiple, contract addressing + key guards), `parseGasToRaw` (×2, mainnet spend), and `trimString` (×45). The danger is not the duplication but the _subtle_ divergence (strict vs coercing `trim`, throw vs empty-on-invalid, rounding/rejection rules). Consolidate, but pick the semantics deliberately per call site — several of these have a load-bearing "this copy throws / that copy returns empty" difference.

5. **God-files / monolithic functions on the riskiest paths.** `crypto.js` (793 lines), `persistence.js` (713), `prepareOracleFulfillment` (295), `handleOracleFeed`, `executeBuiltinCompute` (19-arm switch), the dashboard tabs (900+ lines), and `expireStaleRequest`. The pattern is mixed concerns (parsing + policy + transport + state mutation) that force re-review of high-trust logic on every edit. Extract pure, independently-testable seams — but for the consensus/security-critical ones (signing tail, attestation gates, ticket builders), land the tests _with_ the extraction, since several of these paths have no current coverage.

6. **Dead/divergent ABIs and interface contracts.** The dead `onMiniAppResult` callback (Neo), the hand-synced EVM `Request` tuple, and the divergent EVM grant-key derivation all advertise or depend on a contract that the producer never honors or that lives in two places. These convert a compile-time guarantee into a silent runtime corruption. Single-source the type/interface so drift becomes a build error.

7. **Error classification by free-text substring.** The relayer's transport-vs-deterministic decision (`isNeoXTransportError`, `classifyError`) and the Supabase quota-vs-connectivity backoff selector both infer intent from English phrasing — a node rewording a message can finalize a recoverable request permanently or pick the wrong backoff window. Move to structured codes/discriminants at the seam where the error originates, while keeping a narrow text fallback for foreign/cross-process errors (it can't be deleted).
